🧠 Bellek Modeli — Süreç Adres Alanı

Bir sürecin sanal adres alanı 4 ana bölgeye ayrılır. Stack aşağı doğru büyür (yerel değişkenler, fonksiyon çağrıları), Heap yukarı doğru büyür (malloc/new). Aralarında büyüme payı vardır. Data segment global/static değişkenleri, Text segment ise makine kodunu içerir.

👉 İşaretçi (Pointer) Simülatörü

İşaretçi, başka bir değişkenin bellek adresini tutan değişkendir. int x = 42; bellekte bir yerde saklanır. int *p = &x; ise x'in adresini tutar. *p ile bu adresteki değere erişilir (dereferencing). Aşağıda bellek hücrelerini ve işaretçi ilişkilerini interaktif olarak inceleyin.

📚 Çağrı Yığını (Call Stack) — Stack Frame'ler

Her fonksiyon çağrısı stack'te bir stack frame oluşturur. Frame; dönüş adresi, parametreler, yerel değişkenler ve saklanan register'ları içerir. Fonksiyon döndüğünde frame yok edilir. Stack overflow, çok fazla iç içe çağrı olduğunda (sonsuz özyineleme) stack'in taşmasıdır.

Stack: boş — Çağrı başlatmak için butona tıklayın.

💧 Bellek Sızıntısı (Memory Leak) Simülasyonu

malloc() ile heap'ten ayrılan bellek, işaretçi kaybolduğunda sızıntıya uğrar — bellek artık kullanılamaz ama geri de verilemez. free() çağrısı belleği geri verir. Uzun süre çalışan programlarda biriken sızıntılar sistemin çökmesine yol açar.

Heap boş. malloc() ile bellek ayırın.

🧠 C++ Akıllı İşaretçiler (Smart Pointers)

Modern C++ (C++11 ve sonrası), ham işaretçilerin yol açtığı bellek sızıntılarını ve dangling pointer hatalarını önlemek için akıllı işaretçiler sunar. RAII (Resource Acquisition Is Initialization) prensibiyle bellek otomatik yönetilir.

🔒
unique_ptr
Tek sahiplik (exclusive ownership)
Bir kaynağın sadece bir sahibi olabilir. Kopyalanamaz, sadece move edilebilir. Scope dışına çıkınca otomatik silinir.
auto p = make_unique<int>(42); // p kopyalanamaz auto q = move(p); // sahiplik q'ya geçti // p artık nullptr
👥
shared_ptr
Paylaşımlı sahiplik (reference counting)
Birden çok işaretçi aynı kaynağı paylaşabilir. Referans sayacı sıfırlanınca kaynak otomatik silinir. Döngüsel referanslar sızıntıya yol açar.
auto p = make_shared<int>(42); auto q = p; // ref count = 2 p.reset(); // ref count = 1 q.reset(); // ref count = 0 → silindi
👀
weak_ptr
Zayıf referans (no ownership)
shared_ptr'in işaret ettiği nesneyi sahiplenmeden gözlemler. Döngüsel referansları kırmak için kullanılır. Kullanmadan önce lock() ile shared_ptr'e dönüştürülmelidir.
auto s = make_shared<Node>(); weak_ptr<Node> w = s; if (auto sp = w.lock()) { // sp geçerli, kullanabiliriz } // sp burada yok olur

🔄 Endianness — Byte Sıralaması

Çok baytlı sayılar bellekte iki farklı sırada saklanabilir. Little-endian (x86, ARM): en önemsiz byte en düşük adreste. Big-endian (ağ protokolleri, bazı gömülü sistemler): en anlamlı byte en düşük adreste. Ağ programlamada htonl()/ntohl() ile dönüşüm yapılır.

0x0A0B0C0D sayısının bellekteki dizilimi:

0C
0D
0x104
En anlamlı byte (0A) en düşük adreste
