📜 Soyutlamanın Evrimi: Makine Dili → Assembly → C

Bilgisayarlar sadece 0 ve 1'leri anlar. Assembly, bu ikili kodlara insan tarafından okunabilir sembolik isimler verdi. C ise assembly'nin gücünü korurken taşınabilirlik, tip güvenliği ve yapısal programlama ekledi. Her aşamaya tıklayarak detayları görün.

🔬 Aynı Fonksiyon, Üç Seviye

Bir sayı listesindeki en büyük değeri bulan fonksiyonun makine dili (hex opcode), x86-64 assembly ve C karşılaştırması.

⚡ Makine Dili (x86-64 Hex)
48 89 F8 ; mov rax, rdi 48 85 F6 ; test rsi, rsi 7E 12 ; jle .done 31 C9 ; xor ecx, ecx 8B 14 8F ; mov edx, [rdi+rcx*4] 39 D0 ; cmp eax, edx 0F 4D C2 ; cmovge eax, edx FF C1 ; inc ecx 39 F1 ; cmp ecx, esi 7C F3 ; jl .loop C3 ; ret
Her byte bir komut veya operand. CPU bu byte'ları decode edip çalıştırır. Hata ayıklamak neredeyse imkansız.
🔧 x86-64 Assembly (AT&T)
max_value: movq %rdi, %rax # max = arr[0] testq %rsi, %rsi # len == 0? jle .done # evetse bitir movl $1, %ecx # i = 1 .loop: movl (%rdi,%rcx,4), %edx cmpl %edx, %eax cmovll %edx, %eax # if(arr[i]>max) incq %rcx # i++ cmpq %rsi, %rcx jl .loop .done: ret
Sembolik isimler (max_value, .loop), register isimleri, label'lar. Assembler bunu makine diline çevirir.
💻 C Dili
int max_value(int *arr, int len) { if (len <= 0) return 0; int max = arr[0]; for (int i = 1; i < len; i++) { if (arr[i] > max) max = arr[i]; } return max; }
İnsan tarafından okunabilir. Tip güvenliği, yapısal bloklar (if, for), değişken isimleri. Derleyici bunu optimize assembly'ye çevirir.

🧪 C → Assembly Etkileşimli Gezgini

C kodunun derleyici tarafından nasıl assembly'ye dönüştürüldüğünü görün. GCC -O2 optimizasyon seviyesini simüle eden basitleştirilmiş bir dönüşüm. C kodunu değiştirin ve karşılık gelen x86-64 assembly çıktısını inceleyin.

🗂️ x86-64 Register Ailesi

x86-64 mimarisinde 16 genel amaçlı register, RIP (instruction pointer), RFLAGS ve segment register'ları bulunur. Register'lar tarihsel olarak birikmiştir: rax (64-bit) → eax (32-bit) → ax (16-bit) → al/ah (8-bit). Her register'a tıklayarak görevini görün.

📞 Çağrı Düzeni (Calling Convention) — System V AMD64 ABI

Fonksiyonlar arası geçişte parametrelerin nasıl aktarılacağını, dönüş değerinin nerede olacağını ve hangi register'ların korunması gerektiğini belirleyen kurallar bütünüdür. Linux/macOS'ta System V AMD64 ABI kullanılır.
